# Legal Protection Summary

This document summarizes the legal protections in place for the Solcial project.

## License Type

**Proprietary License (Source-Available)**

The project uses a custom proprietary license that makes the source code visible but restricts usage.

## Key Legal Documents

### 1. LICENSE
- Full legal terms and conditions
- Defines what users can and cannot do
- Grants limited rights for viewing and contributing only
- Explicitly prohibits commercial use, deployment, and distribution

### 2. NOTICE
- Copyright notice
- Additional legal warnings
- Trademark information
- Contact information for licensing inquiries

### 3. README.md
- Prominent warning at the top
- Clear explanation of restrictions
- License summary for quick reference

### 4. CONTRIBUTING.md
- Contribution License Agreement (CLA)
- Explains that contributions become property of Solcial Team
- Clarifies that the project is NOT open source

### 5. GitHub Templates
- Pull Request template with license agreement checkbox
- Issue templates with license acknowledgment
- Ensures contributors are aware of terms

## What Users CAN Do

✅ View the source code for educational purposes
✅ Fork the repository to submit contributions
✅ Submit pull requests to improve the project
✅ Study the code to learn

## What Users CANNOT Do

❌ Use the software for commercial purposes
❌ Deploy or run the software in production
❌ Create derivative works for distribution
❌ Clone or fork for any purpose other than contributing
❌ Remove or alter copyright notices
❌ Use the project's name or branding without permission

## Enforcement

### Automatic Protections
1. **Copyright**: Automatically protects the code
2. **License Terms**: Legally binding contract
3. **Contribution Agreement**: Contributors grant rights to maintainers

### Manual Enforcement
If someone violates the license:
1. Send a cease and desist letter
2. Request takedown from hosting platforms (GitHub, etc.)
3. Pursue legal action if necessary

## Commercial Licensing

For businesses or individuals who want to:
- Use the software commercially
- Deploy in production
- Create derivative works
- Distribute the software

Contact: team@solcial.app

## Best Practices

### For Repository Owners
1. ✅ Monitor for unauthorized forks/clones
2. ✅ Review all contributions carefully
3. ✅ Respond quickly to license violations
4. ✅ Keep license and legal documents up to date
5. ✅ Document all commercial licensing agreements

### For Contributors
1. ✅ Read and understand the LICENSE before contributing
2. ✅ Only fork for the purpose of contributing
3. ✅ Delete your fork after PR is merged
4. ✅ Don't use any code from this project in your own projects
5. ✅ Respect the proprietary nature of the software

## Additional Protections

### Package.json
- `"license": "UNLICENSED"` - Indicates proprietary software
- `"private": true` - Prevents accidental npm publishing

### Git History
- All commits are tracked
- Contribution history is preserved
- Proves ownership and timeline
